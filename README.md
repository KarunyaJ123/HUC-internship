# Git and GitHub Basics 

## What is Git?
Git is a version control system used to track changes in files and source code. It helps developers save different versions of their work and return to earlier versions when required.

## What is GitHub?
GitHub is an online platform that hosts the Git repositories. It allows users to store their projects on the internet and share code with others.

## Why Git and GitHub are used?
Git helps in managing the code efficiently and prevents data loss. GitHub makes collaboration easy and allows developers to work together from different locations.

## Differencs between Git and GitHub
* Type - Git is a tool/software.
         GitHub is a website/platform.
* Definition - Git is a version control system.
               GitHub is a web-based hosting platform for Git repositories.
* Purpose - Git tracks changes in source code.
            GitHub stores, and manages Git repositories online.
* Usage - Git used to commit, branch and merge code.
          GitHub used to collaborate, review code, and manage projects.
* Ownership - Git is Open-source tool.
              GitHub owned by a company.
* Examples - Git -> git commit, git branch
             GitHub -> pull request, issues, forks

## Basic Git Workflow
1. Write or modify code files
2. Commit the changes
3. Push the changes to GitHub
4. Pull updates when necessary

## How GitHub helps in Team Collaboration?
GitHub supports teamwork by allowing multiple developers to work on the same project using branches and pull requests. Team members can review code, suggest improvements, and merge changes safely.
